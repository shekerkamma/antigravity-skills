# Agent Spec: customer-support-agent

## Overview
A graph workflow agent using ADK 2.0 that acts as a customer support representative for a shipping company. 

It implements the following logic:
1. **Classification Node**: Takes the user's query and classifies it as either "shipping-related" (rates, tracking, delivery, returns) or "unrelated".
2. **Router**: Routes the execution flow based on the classification.
3. **Shipping FAQ Agent Node**: If the query is related to shipping, this LLM agent answers the query.
4. **Decline Node**: If the query is unrelated, this node politely declines to answer.

## Example Use Cases
- **Shipping Query**: 
  - Input: "What are your rates for overnight delivery to Chicago?"
  - Output: Accurate shipping rates response from the Shipping FAQ agent.
- **Unrelated Query**:
  - Input: "Can you recommend a good recipe for apple pie?"
  - Output: "I apologize, but I can only assist with shipping-related queries like rates, tracking, delivery, and returns. How can I help you with your shipment today?"

## Tools Required
None needed for this prototype workflow (uses standard LLM classification and FAQs).

## Constraints & Safety Rules
- The agent must only answer shipping-related queries (rates, tracking, delivery, returns).
- It must politely decline to answer unrelated queries.
- It must not use placeholders or mock information that could confuse users.

## Success Criteria
- Correct routing of shipping queries to the shipping FAQ agent.
- Correct routing of unrelated queries to the decline node.
- Fluent, polite responses from both paths.
